## 问题 2：Isaac Lab API 版本不兼容（`dump_pickle`）

### 现象

- 训练脚本报错：`cannot import name 'dump_pickle' from isaaclab.utils.io`

### 原因

- 当前本机 Isaac Lab 版本中，`isaaclab.utils.io` 不再提供 `dump_pickle`。

### 解决

- 修改 `scripts/skrl/train.py`：
  - `from isaaclab.utils.io import dump_pickle, dump_yaml` 改为仅导入 `dump_yaml`
  - 用 Python 标准库 `pickle.dump(...)` 保存 `env.pkl / agent.pkl`

### 具体代码修改

- 文件：`scripts/skrl/train.py`
- 关键改动 1（导入）：

```python
# 修改前
from isaaclab.utils.io import dump_pickle, dump_yaml

# 修改后
from isaaclab.utils.io import dump_yaml
import pickle
```

- 关键改动 2（保存配置）：

```python
# 修改前
dump_pickle(os.path.join(log_dir, "params", "env.pkl"), env_cfg)
dump_pickle(os.path.join(log_dir, "params", "agent.pkl"), agent_cfg)

# 修改后
with open(os.path.join(log_dir, "params", "env.pkl"), "wb") as f:
    pickle.dump(env_cfg, f)
with open(os.path.join(log_dir, "params", "agent.pkl"), "wb") as f:
    pickle.dump(agent_cfg, f)
```

## 问题 3：动态库冲突（`CXXABI_1.3.15`）

### 现象

- GUI 启动时扩展导入报错：`libstdc++.so.6: version 'CXXABI_1.3.15' not found`

### 原因

- Isaac Sim 与 Conda 环境动态库加载顺序冲突，运行时没有优先使用 Conda 内匹配的 `libstdc++`。

### 解决

- 启动前导出：

```bash
export LD_LIBRARY_PATH=/home/ylee/anaconda3/envs/env_isaaclab/lib:${LD_LIBRARY_PATH}
```

### 具体修改说明

- 该问题未改仓库代码，仅调整运行命令环境变量。

## 问题 4：机器人资产路径错误（USD 文件不存在）

### 现象

- 环境构建时报错：`USD file not found ... Sim-Circular Cartpole L1.SLDASM.usd`

### 原因

- 项目配置引用了 `.usd` 路径，但仓库中实际存在的是 `.urdf` 文件。

### 解决

- 修改 `circular_cartpole_robot_cfg.py`：
  - 将 `UsdFileCfg(usd_path=...)` 改为 `UrdfFileCfg(asset_path=...)`
  - 路径改为存在的：
    - `models/.../Sim-Circular Cartpole L1.SLDASM.urdf`
    - `models/.../Sim-Circular Cartpole L2.SLDASM.urdf`
    - `models/.../Sim-Circular Cartpole L3.SLDASM.urdf`

### 具体代码修改

- 文件：`source/circular_cartpole/circular_cartpole/tasks/manager_based/circular_cartpole/circular_cartpole_robot_cfg.py`
- 关键改动 1（路径变量）：

```python
# 修改前（示例）
l1_usd_path = os.path.join(os.getcwd(), ".../Sim-Circular Cartpole L1.SLDASM.usd")

# 修改后
l1_urdf_path = os.path.join(os.getcwd(), "models/Sim-Circular Cartpole L1.SLDASM/urdf/Sim-Circular Cartpole L1.SLDASM.urdf")
```

- 关键改动 2（加载方式）：

```python
# 修改前
spawn=sim_utils.UsdFileCfg(
    usd_path=l1_usd_path,
    ...
)

# 修改后
spawn=sim_utils.UrdfFileCfg(
    asset_path=l1_urdf_path,
    fix_base=True,
    ...
)
```

## 问题 5：URDF 配置校验缺省值（`joint_drive.gains.stiffness`）

### 现象

- 报错：`Missing values ... scene.robot.spawn.joint_drive.gains.stiffness`

### 原因

- `UrdfFileCfg` 需要显式提供 `joint_drive` 的增益字段，默认值在当前版本校验下不满足。

### 解决

- 在 `UrdfFileCfg` 中补充：

```python
joint_drive=sim_utils.UrdfConverterCfg.JointDriveCfg(
    gains=sim_utils.UrdfConverterCfg.JointDriveCfg.PDGainsCfg(stiffness=0.0, damping=0.0)
)
```

### 具体代码修改

- 文件：`source/circular_cartpole/circular_cartpole/tasks/manager_based/circular_cartpole/circular_cartpole_robot_cfg.py`
- 在 `CIRCULAR_CARTPOLE_ROBOT_L1_CFG / L2_CFG / L3_CFG` 的 `UrdfFileCfg(...)` 中都新增了：

```python
joint_drive=sim_utils.UrdfConverterCfg.JointDriveCfg(
    gains=sim_utils.UrdfConverterCfg.JointDriveCfg.PDGainsCfg(stiffness=0.0, damping=0.0)
)
```
